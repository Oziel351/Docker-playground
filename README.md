# Docker-playground

This repository is a personal learning workspace focused on Docker.

The goal of this project collection is to demonstrate the knowledge I have gained while learning Docker, from basic concepts to more advanced topics, by building and documenting different hands-on projects over time. Each project in this repository has its own purpose, architecture, challenges, and solution path.

Every project includes:

- its own `README.md` explaining what it is about
- the main Docker concepts used
- the problems encountered and how they were solved
- a diagram stored in the corresponding `docs/` section

## Purpose

This repository is not a single production application. It is a playground for experimenting with Docker concepts, reinforcing what I learn, and documenting each step clearly as I move through different projects.

The topics covered across the projects may include:

- Docker images and containers
- Dockerfiles and multi-stage builds
- Docker Compose
- networking between containers
- volumes and persistence
- environment variables and secrets
- health checks and service dependencies
- service isolation and best practices
- frontend, backend, database, and cache containers

## Projects

Each project lives in its own folder and contains a self-contained explanation of the stack and implementation.

- [docker-fullstack](docker-fullstack/README.md)

## Documentation

Visual diagrams and supporting material are stored inside each project under its own `docs/` directory.

## Learning Approach

The idea behind this repository is to learn by building, testing, breaking, and fixing.  
Each project is used as a practical example of a Docker concept, and the documentation explains both the final solution and the reasoning behind it.

## Notes

- This repository evolves as new Docker concepts are learned.
- Older projects may reflect earlier stages of learning.
- The README of each project contains the detailed explanation for that specific case.
